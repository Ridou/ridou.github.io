Inheritance vs Composition

I like options

Classical inheritance is defined by the relationship between two objects mostly known as the superclass and subclass. Subclasses are specializations of their superclasses. Thus, creating the generalization/specialization relationship. Inheritance solves the problem of objects that share common behavior and allows to isolate shared code to implement those common algorithms into a subclass, while also allowing the possibility of adding new subclasses. Using super you can hook all the variables from the superclass into a subclass

Composition is similar to inheritance but instead of having to rely on implicit relationship of superclass/subclass you alter or replace functions in a module. It allows you to call functions in other classes.

In my opinion inheritance makes it much more easier to do open source coding because it allows the new user to add an extension with ease. Both composition and inheritance allow the user to reuse lines of code. I would use composition to package up code into modules that are used in many unrelated situations and inheritance when I am working with a common concept.
-Richard Santin
